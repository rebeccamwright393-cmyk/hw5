TOOLS FOR YOUTUBE CHANNEL JSON
═══════════════════════════════════════════════════════════════════════════════

generateImage
- Purpose: Generate an image from a text prompt. Optionally uses an anchor/reference image the user has dragged in (e.g. "make this a YouTube thumbnail with bold text"). The generated image is displayed in chat, can be clicked to enlarge (lightbox), and downloaded.
- Input schema: { "prompt": "string" (required), "anchorImage": "string" (optional, base64) }
- Output schema: { "imageData": "string" (base64), "mimeType": "string" } or { "error": "string" }
- Example prompts: "Generate an image of a sunset over mountains", "Create a YouTube thumbnail with bold text and high contrast", "Generate like this but with a beach", "Make an image in this style with my cat"

plot_metric_vs_time
- Purpose: Plot a numeric field (view_count, like_count, comment_count) over time. Sorts by release_date when available; otherwise uses video order. Renders an interactive line chart in chat (click to enlarge, download as SVG/PNG).
- Input schema: { "field": "string" } — e.g. "view_count", "like_count", "comment_count"
- Output schema: { "_plotMetricVsTime": true, "field": "string", "data": [{ "name", "value", "rawDate" }] } or { "error": "string" }
- Example prompts: "Plot view_count vs time", "Chart likes vs release date", "Graph comment_count vs time", "Show me views over time"

play_video
- Purpose: When the user asks to play or open a video from the loaded channel JSON, resolve which video they mean and return a clickable card with title, thumbnail, and URL that opens in a new tab.
- Input schema: { "query": "string" } — title keyword, ordinal ("first", "second", "3rd"), or "most viewed"
- Output schema: { "title": "string", "thumbnailUrl": "string", "videoUrl": "string" } or { "error": "string" }
- Example prompts: "Play the video about asbestos", "Open the first video", "Play the most viewed video", "Show me the second video"

compute_stats_json
- Purpose: Compute descriptive statistics (count, mean, median, std, min, max) for a numeric field across all videos in the loaded channel JSON.
- Input schema: { "field": "string" } — e.g. "view_count", "like_count", "comment_count", "duration"
- Output schema: { "field": "string", "count": number, "mean": number, "median": number, "std": number, "min": number, "max": number } or { "error": "string" }
- Example prompts: "What's the average view count?", "Give me stats on like_count", "Summary of comment_count", "Compute stats for view_count", "Mean and median for duration"

CSV TOOLS (when user uploads a CSV)
═══════════════════════════════════════════════════════════════════════════════

When a CSV is uploaded with Favorite Count and View Count columns, an "engagement" column is automatically added: engagement = Favorite Count / View Count.

- compute_column_stats(column): mean, median, std, min, max for a numeric column
- get_value_counts(column, top_n): count of each unique value in a categorical column
- get_top_tweets(sort_column, n, ascending): top or bottom N tweets sorted by any metric (including "engagement")

When a CSV is present, prefer these tools for simple stats. Use code execution (Python) only when the user asks for plots or analysis the tools above can't handle.
